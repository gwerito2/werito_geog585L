Question 1

What request type is common across all three (WMS, WFS, WCS) OGC web services that we have studied?

Answerr:

GetCapabilities is most common of all three OGC web services.

Question 2

A What are the required parameters, and what do they represent?

B What is returned in response to a WMS GetCapabilities request?

Answers:

A The required parameters are "SERVICE=WMS" and "REQUEST=GetCapabilities".

B It returns the "Service Metadata" XML file representing information about specific WMS service and its layers.

Question 3

A What are the required parameters, and what do they represent?

B What is returned in response to a WMS GetMap request?

C What is the significance of transparency in WMS requests?

Answers:

A The required parameters and representatives are:

  • VERSION=version, Request version.
  • REQUEST=GetMap, Request name.
  • LAYERS=layer_list, Comma-separated list of one or more map layers.
  • STYLES=style_list, Comma-separated list of one rendering style per requested layer.
  • SRS=namespace:identifier, Spatial Reference System.
  • BBOX=minx,miny,maxx,maxy, Bounding box corners (lower left, upper right) in SRS units.
  • WIDTH=output_width, Width in pixels of map area.
  • HEIGHT=output_height, Height in pixels of map area.
  • FORMAT=output_format, Output format of map type.

B It returns a "Map Images" representing one or more layers from a WMS serviced area.

C The significance of transparency allows the layer to be used as an overlay, while being able to show the underlying layers.

Question 4

What OGC request would you use to inform the configuration of a client application (like ArcGIS) about an OGC service that you want to add layers from?

Answer:

The GetCapabilites request would be used to inform the client's application and what the service may offer.
